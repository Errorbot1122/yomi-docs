.. _class_actionuidata_method_get_data:

.. rst-class:: classref-method

:ref:`Dictionary<class_dictionary>` **get\_data**\ \(\ \ \) :ref:`🔗<class_actionuidata_method_get_data>`

Called by the Action Menu when the player has locked in their selection, or when the prediction resets after  :ref:`^data_changed()<class_actionuidata_signal_data_changed>`  is emitted.

The returned  :ref:`^Dictionary<class_dictionary>`  is provided to the state that owns this  :ref:`^ActionUIData<class_actionuidata>`  and is available in it's script via it's  :ref:`^StateInterface.data<class_stateinterface_property_data>`  variable during the states execution.

By default, depending on how many child nodes this  :ref:`^ActionUIData<class_actionuidata>`  contains, the data returned is either the data from the child nodes  :ref:`^get_data()<class_actionuidata_method_get_data>`  function directly if there is only one child, or if there are two or more children it will be a  :ref:`^Dictionary<class_dictionary>`  of each child nodes  :ref:`^get_data()<class_actionuidata_method_get_data>`  values organized by child nodes names.



.. code-block:: gdscript

    
    # Example for one 8Way child
    data["x"] # or data.x
    data["y"] # or data.y
    
    # Example for an 8Way named "Direction" and a Slider named "Distance"
    data["Direction"]["x"] # or data.Direction.x\ndata["Direction"]["y"] # or data.Direction.y
    data["Distance"]["x"] # or data.Distance.x
