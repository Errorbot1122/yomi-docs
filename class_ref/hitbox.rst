.. _class_hitbox_property_block_cancel_allowed:

.. rst-class:: classref-property

:ref:`bool<class_bool>` **block\_cancel\_allowed** = ``true`` :ref:`🔗<class_hitbox_property_block_cancel_allowed>`

.. rst-class:: classref-item-separator

----


.. _class_hitbox_property_block_punishable:


:badge-export:`@export` 


.. rst-class:: classref-property

:ref:`bool<class_bool>` **block\_punishable** = ``false`` :ref:`🔗<class_hitbox_property_block_punishable>`

If ``false``, hitting the opponent while they are soft blocking *(without a **Perfect Parry**)* will cause the player to gain their next interrupt turn on their opponents next turn. This generally means that if ``true``, the player will not get a turn when hitting the opponents block.

Has no effect if  :ref:`^block_cancel_allowed<class_hitbox_property_block_cancel_allowed>`  is ``false``, or if the parent  :ref:`^CharacterState.interruptible_on_opponent_turn<class_characterstate_property_interruptible_on_opponent_turn>`  is ``true``.

The amount of ticks the start of the opponents next state will be delayed for after hitting them while they are soft blocking *(without a **Perfect Parry**)*.

Negative values will cause the attacker to be delayed instead.

***Tip:** values should generally range from -1 to +4 depending on speed, size, and other aspects, while projectiles hitboxes should generally be +2. You can always check the base roster for "balancing advice"*"
